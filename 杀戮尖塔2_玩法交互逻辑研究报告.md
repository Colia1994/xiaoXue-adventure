## 杀戮尖塔2（Slay the Spire 2）玩法交互逻辑研究报告

### 概述

《杀戮尖塔2》由Mega Crit开发，2025年3月在Steam开启抢先体验。作为卡牌构筑Roguelike鼻祖的续作，它继承了一代的核心回合制卡牌战斗+地图探索框架，同时引入了附魔系统、远古者(Ancients)、任务卡(Quests)、交替章节(Alternate Acts)、联机合作等全新机制，大幅拓展了策略深度和重玩价值。引擎从LibGDX迁移到Godot，画面和交互体验有明显升级。

---

### 一、核心战斗系统

#### 1.1 回合制能量制战斗

战斗采用回合制，基本流程为：

- **回合开始**：从抽牌堆抽取5张牌（默认），获得3点能量（默认）。
- **玩家行动**：打出手牌消耗对应能量，攻击牌造成伤害，技能牌产生格挡/增益/特殊效果，能力牌(Power)提供持久被动效果。
- **回合结束**：未使用手牌进入弃牌堆（标记"保留/Retain"的卡牌除外，可留至下回合）。
- **敌人行动**：敌人按预示意图(Intent)执行攻击/防御/施加Debuff。
- **牌堆循环**：抽牌堆耗尽后，弃牌堆洗入抽牌堆重新抽取。

#### 1.2 敌方意图预示

敌人头上的图标预示下一步动作——攻击（含伤害数值）、防御、施加负面状态、强化自身等。玩家需根据预示信息合理分配攻防资源。

#### 1.3 卡牌类型

- **攻击牌(Attack)**：造成伤害，部分附带额外效果（抽牌、弃牌、施加Debuff）
- **技能牌(Skill)**：提供格挡(Block)、增益、操控牌堆等
- **能力牌(Power)**：打出后永久生效，提供每回合被动效果
- **状态牌(Status)**：负面卡牌，塞入牌组降低效率（不可打出或打出有负面效果）
- **诅咒牌(Curse)/苦难(Affliction)**：二代新增的负面卡牌类型，比一代诅咒更多样化

#### 1.4 关键战斗状态

力量(Strength)增加攻击伤害、敏捷(Dexterity)增加格挡值、易伤(Vulnerable)使受到伤害增加50%、虚弱(Weak)使攻击伤害降低25%、中毒(Poison)每回合扣血等。

### 二、卡牌系统与附魔机制

#### 2.1 卡牌获取与构筑

- 战斗胜利后从3张随机牌中选取1张（或跳过）
- 商店购买特定卡牌
- 事件(?)房间随机获取
- 卡牌有普通/罕见/稀有三个稀有度等级

#### 2.2 卡牌升级

在休息点(Rest Site)选择"打铁/Smith"可升级一张牌，提升数值或追加效果。升级后牌名后带"+"标记。

#### 2.3 附魔系统（Enchantments）—— 二代核心新增

附魔是《杀戮尖塔2》最标志性的新机制，为卡牌自定义提供了全新维度：

- **获取方式**：通过遗物效果、事件选项、远古者赐福获得
- **规则**：每张卡牌仅能拥有一种附魔，不可叠加同名附魔，一旦附魔不可移除
- **独立于升级**：附魔和升级是两套独立系统，一张牌可以同时被升级和附魔
- **效果举例**：
  - "Adroit"（灵巧）—— 打出时额外获得格挡
  - "Clone"（克隆）—— 在休息点可复制该卡牌
  - "Soulbound"（灵魂绑定）—— 特定角色专属增强效果

附魔机制让每次Run中即使获得相同卡牌也会有不同的构筑方向，极大增加了策略多样性。

### 三、角色系统（5个可操作角色）

抢先体验版开放5个角色，其中3个从一代回归但全面重做卡组，2个全新角色。

#### 3.1 铁甲战士（Ironclad）

- **初始HP**：80，自带遗物"燃烧之血"（每场战斗结束回复6HP）
- **核心机制**：力量叠加 + 格挡协同，攻防一体
- **卡组风格**：强化自身力量提升攻击伤害，同时有大量格挡手段。新手最友好。

#### 3.2 静默猎人（The Silent）

- **定位**：敏捷型刺客，灵活输出
- **核心机制**：毒流（累积中毒持续掉血）、小刀流（生成0费Shiv爆发输出）、弃牌流
- **特点**：大量过牌和爆发卡牌，循环逻辑优化，操作上限高

#### 3.3 故障机器人（Defect）

- **核心机制**：电球(Orb)系统——闪电/冰霜/暗/等离子四类电球，每回合被动触发效果
- **卡组风格**：物理流与状态流，通过电流相互协同产生复杂连锁效果

#### 3.4 储君/摄政王（The Regent）—— 全新角色

- **核心机制**：锻造(Forge)系统——固有牌"主权之刃"(Sovereign Blade)为2费攻击牌，每场战斗开局握在手中，通过锻造持续强化其数值
- **星辰(Stars)系统**：专属资源，用于触发特殊效果
- **能量保留**：部分机制允许未使用能量跨回合保留
- **特点**：前期偏弱，需要时间"铸剑"积累，后期爆发极强

#### 3.5 亡灵契约师（Necrobinder）—— 全新角色

- **初始HP**：66，自带遗物"缚魂命匣"（每回合开始给同伴奥斯提回复1HP）
- **核心机制**：
  - **召唤系统**：拥有独立血条的同伴"奥斯提"(Osthi)，可通过召唤增加其HP或令其复活
  - **灵魂(Soul)资源**：专属资源体系，用于联动过牌、回复、触发卡牌效果
  - **灾厄(Affliction)机制**：利用回合结束结算机制直接"斩杀"敌人
- **特点**：强防御，独特的双血条运营策略

---

### 四、远古者系统（Ancients）—— 替代Boss遗物

#### 4.1 机制说明

远古者系统替代了一代的Boss遗物选择：

- **触发时机**：每一章(Act)开始时遭遇远古者
- **强制选择**：玩家不能跳过，必须从远古者提供的选项中选择一个赐福
- **首章固定**：第一章必定遭遇"涅奥"(Neo)
- **赐福池**：共8位远古者，每位有不同的遗物/卡牌池，部分需要通过Darv解锁
- **不可逆**：选择后不可撤销

#### 4.2 设计意图

远古者使每次Run的开局策略更加多样化——你无法"跳过"这个选择，必须根据当前牌组状态做出适应性决策。部分赐福有时限（如"玩具盒"三场后消失），增加了时间压力和策略深度。

---

### 五、地图探索与事件系统

#### 5.1 地图结构

- 一局游戏分为**三个章节(Act)**，每个章节有独立地图
- 每个章节有唯一起始房间和结束房间（Boss），中间为分支路径
- 玩家选择路线，路径一旦选定不可回头

#### 5.2 房间类型

| 房间类型 | 功能 |
|---------|------|
| 普通战斗 | 遭遇普通怪物，胜利获取金币+选卡 |
| 精英怪(Elite) | 强力敌人，额外掉落遗物 |
| 休息点(Rest Site) | 选择：休息(恢复HP) / 打铁(升级卡牌) |
| 未知房间(?) | 随机事件，可能获得卡牌/遗物/金币/负面效果 |
| 商店(Shop) | 购买卡牌/遗物/药水，花费金币移除卡牌 |
| Boss | 章节最终Boss，击败后获得大量奖励并进入下一章节 |

#### 5.3 任务卡系统（Quest Cards）—— 二代新增

任务卡是一种不可打出的特殊卡牌，从"?"房间获取：

- 获取后占用牌组空间（但不可打出，不消耗手牌位）
- 需满足特定条件完成任务
- 完成后获得强力奖励

**已知任务示例**：
- "Byrdonis Egg"（鸟巢蛋）：在休息点孵化，获得0费14伤害攻击牌+同伴
- "Spoils Map"（战利品地图）：携带至下一章节，获得600金币
- "Lantern Key"（灯笼钥匙）：击败特定精英怪Flail Knight后，选择History Course遗物或第三章宝箱

#### 5.4 交替章节（Alternate Acts）—— 二代新增

引入分支剧情路径，不同Run可能进入不同的章节变体，改变每次冒险的整体结构。增强了Roguelike的新鲜感和重玩动力。

#### 5.5 事件系统

游戏内有50+事件，在"?"房间触发。事件通常提供多个选项，涉及权衡取舍（如：用HP换取强力卡牌，或安全跳过）。

### 六、遗物系统

#### 6.1 遗物获取

- 精英怪击败后掉落
- 商店购买
- 事件奖励
- 远古者赐福

#### 6.2 遗物分级

- **起始遗物(Starter)**：每个角色自带一个（如铁甲战士的"燃烧之血"）
- **普通遗物(Common)**
- **罕见遗物(Uncommon)**
- **稀有遗物(Rare)**
- **远古遗物(Ancient)**：来自远古者系统，替代一代Boss遗物

#### 6.3 遗物交互

遗物提供全局被动效果，贯穿整个Run。与卡组构筑形成协同——例如某遗物"每打出3张攻击牌抽1张"会鼓励构建低费攻击流。

---

### 七、合作模式（Co-op）—— 二代全新

#### 7.1 基本规则

- 支持最多**4人**在线联机合作
- **共享**：地图路线、敌人身上的Buff/Debuff
- **独立**：每人有独自的卡组、生命值、遗物、能量
- **角色不重复**：每位玩家必须选择不同角色

#### 7.2 战斗机制

- 使用共享回合制(Shared Turn)，所有玩家在同一回合内行动
- 战斗中Debuff在敌人身上由全队共享（如一人施加易伤，全队打出的伤害都享受加成）
- Boss战全队汇合

#### 7.3 地图交互

- 分支节点可以分头行动，各自走不同路线
- Boss节点强制汇合
- 新增团队专属卡牌与遗物

#### 7.4 难度调整

多人模式下怪物属性有所增强（血量、伤害），以平衡多人输出优势。通关多人模式可解锁专属内容。

---

### 八、药水系统

- 药水在战斗中使用，不消耗能量和回合
- 携带槽位有限（默认2-3格）
- 获取途径：战斗掉落、商店购买、事件获取
- 药水效果包括：回复HP、临时增加力量/敏捷、造成伤害、抽牌等

---

### 九、交互流程总结

一次完整Run的交互循环为：

```
选择角色 → 遭遇远古者(选赐福) → 地图选路 → 进入房间
  ↓
[战斗房间] → 回合制卡牌战斗 → 胜利奖励(选卡/金币)
[事件房间] → 阅读描述 → 选择选项 → 获得/失去资源
[休息点]   → 选择休息/升级/特殊选项
[商店]     → 购买卡牌/遗物/药水/移除卡牌
  ↓
重复直到Boss → 击败Boss → 下一章节远古者 → 循环
  ↓
第三章Boss通关 = 胜利
```

### 十、相较一代的核心变化总结

| 维度 | 一代 | 二代 |
|------|------|------|
| 角色数量 | 4个 | 5个（3回归+2新） |
| 卡牌强化 | 仅升级 | 升级 + 附魔双系统 |
| Boss遗物 | 击败Boss后三选一（可跳过） | 远古者系统，强制选择 |
| 联机 | 无 | 最多4人合作 |
| 地图结构 | 固定3章 | 3章 + 交替章节变体 |
| 任务系统 | 无 | 任务卡(Quest Cards) |
| 负面卡牌 | 诅咒/状态 | 诅咒/状态 + 苦难(Affliction) |
| 引擎 | LibGDX (Java) | Godot |
